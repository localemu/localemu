# Security Policy

## Reporting a vulnerability

If you believe you have found a security issue in LocalEmu, please report
it privately so it can be fixed before public disclosure.

**Email:** [security@localemu.cloud](mailto:security@localemu.cloud)

Please include in the report:

- A clear description of the issue and the impact you believe it has.
- Steps to reproduce. A minimal repro (one or two commands, a small
  snippet of code, the AWS service involved) is the fastest path to a
  fix.
- The LocalEmu version (`localemu --version`) and how you installed it
  (pip / Docker image tag).
- Your operating system + Python version if relevant.

We aim to acknowledge every report within **2 business days** and to ship
a fix or mitigation within **30 days** for confirmed issues, faster for
anything that exposes user data or enables remote code execution.

## What is in scope

- The LocalEmu core runtime (`localemu/localemu`).
- The published Docker images on Docker Hub.
- The Lambda runtime layer LocalEmu uses to execute user functions.
- The CLI wrappers (`localemu`, `awsemu`).
- The Terraform / CloudFormation export pipeline.

## What is out of scope

- Vulnerabilities in third-party dependencies that LocalEmu pulls in
  unchanged. Please report those to the upstream project; we will
  upgrade when they release a fix.
- Issues that require an attacker to already control the host running
  LocalEmu. LocalEmu is a development tool; we assume the local user
  trusts the local machine.
- Reports generated by automated scanners without a working repro.

## Responsible disclosure

Please give us a reasonable window to fix the issue before publishing
details. We are happy to credit reporters by name (or anonymously, if
you prefer) in the release notes once a fix ships.

## Out-of-band contact

If you cannot reach us at the email above, or the issue is highly
time-sensitive, you can also open a private GitHub Security Advisory
on the [main repository](https://github.com/localemu/localemu/security/advisories/new).
